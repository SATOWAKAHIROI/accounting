# チケット #050: E2Eテスト

## ステータス
- [ ] 未実装

## 機能概要
主要業務フローのエンドツーエンドテスト。

## 実装ファイル
- [ ] `e2e/` ディレクトリ（Cypress or Playwright使用）

## テストツール
- Cypress または Playwright

## テストシナリオ

### シナリオ1: ユーザー登録からログインまで
1. 新規ユーザー登録
2. ログイン
3. ダッシュボード表示確認

### シナリオ2: マスタデータ登録
1. ログイン
2. 会社作成
3. 勘定科目作成（資産、負債、収益、費用）
4. 取引先作成（顧客、仕入先）

### シナリオ3: 仕訳入力から総勘定元帳確認まで
1. ログイン
2. 会計期間作成
3. 仕訳入力（借方・貸方のバランスチェック含む）
4. 総勘定元帳の表示確認
5. 試算表の表示確認（借方合計 = 貸方合計）

### シナリオ4: 請求書作成からPDF出力、入金登録まで
1. ログイン
2. 品目作成
3. 請求書作成（明細行追加、税額計算確認）
4. ステータスを「発行」に変更
5. PDF出力
6. 入金登録
7. 請求書ステータスが「支払済み」に更新されることを確認

### シナリオ5: レポート生成
1. ログイン
2. 損益計算書の表示確認
3. 貸借対照表の表示確認（資産 = 負債 + 純資産）
4. PDF出力
5. Excel出力

### シナリオ6: 会計期間締め処理
1. ログイン
2. 会計期間を締める
3. 締められた期間の仕訳編集が不可であることを確認
4. 期間を再開
5. 仕訳編集が可能になることを確認

## 実行環境
- Docker Composeで全サービスを起動
- テストデータの自動投入
- テスト実行後のデータクリーンアップ
